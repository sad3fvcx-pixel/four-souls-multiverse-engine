# Four Souls Multiverse Engine
# Development Guidelines

Version: 1.0

---

# Цель

Этот документ определяет обязательные правила разработки FSME.

Все изменения движка должны соответствовать данным принципам.

---

# Основные принципы

## 1. Official Rules First

Если существует противоречие между кодом и официальными правилами Four Souls,
правильными считаются официальные правила.

---

## 2. Data Driven

Карты содержат только данные.

Игровая логика реализуется исключительно движком.

---

## 3. No Card Exceptions

Запрещено писать код под конкретную карту.

Если карта не поддерживается,
расширяется DSL или Runtime.

---

## 4. One Source of Truth

Каждая сущность имеет единственное место определения.

Примеры:

- Effect → EFFECT_REGISTRY
- Trigger → TRIGGER_REGISTRY
- Card → CARD_SCHEMA
- Runtime → ENGINE_SPEC

---

## 5. Immutable Definitions

Определения карт, эффектов, триггеров и условий являются неизменяемыми после регистрации.

---

## 6. Explicit over Implicit

Поведение должно быть явно описано.

Запрещены скрытые соглашения.

---

## 7. Small Components

Каждый класс выполняет одну задачу.

---

## 8. Event Driven

Любое изменение GameState происходит через события.

---

## 9. Deterministic

При одинаковом Seed движок обязан воспроизводить абсолютно одинаковую партию.

---

## 10. Test First

Любой новый Effect сопровождается тестами.

---

# Запрещено

- специальная логика для одной карты;
- магические строки;
- глобальное состояние;
- изменение GameState вне Runtime;
- обход Event System;
- обход Stack;
- изменение зарегистрированных определений.

---

# Обязательно

Каждый новый компонент должен ответить на вопросы:

1. Какую проблему он решает?
2. Можно ли решить её существующими средствами?
3. Не нарушает ли он архитектуру?
4. Можно ли покрыть его тестом?
5. Является ли он универсальным?

Если хотя бы на один вопрос ответ "нет",
компонент требует пересмотра.

---

# Definition of Done

Функциональность считается завершённой только если:

- реализована;
- покрыта тестами;
- документирована;
- проходит Replay Tests;
- не нарушает существующие спецификации.
