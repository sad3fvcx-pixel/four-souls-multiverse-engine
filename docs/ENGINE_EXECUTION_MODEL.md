# Four Souls Multiverse Engine
# Engine Execution Model

Version: 1.0

---

# Главный цикл

Пока игра не завершена:

1. Получить следующую команду.
2. Провалидировать команду.
3. Выполнить команду.
4. Сгенерировать события.
5. Найти способности, реагирующие на события.
6. Разрешить способности.
7. Выполнить State-Based Actions.
8. Повторять до стабилизации состояния.
9. Передать управление следующему игроку.

---

# Стабильное состояние

Игровое состояние считается стабильным, если одновременно выполнены условия:

- очередь событий пуста;
- стек пуст;
- отсутствуют ожидающие State-Based Actions;
- отсутствуют обязательные решения игрока.

Только после этого разрешается принять новую команду.

---

# Команда

Любое изменение GameState начинается только с команды.

Команды бывают:

- ActivateItem
- PlayLoot
- AttackMonster
- BuyTreasure
- EndTurn
- ...

---

# Событие

Команда сама ничего не изменяет.

Она лишь инициирует выполнение правил.

После выполнения команды создаются события.

---

# Способность

Каждое событие может активировать способности.

Порядок определяется правилами игры.

---

# Stack

Все стековые эффекты помещаются в Stack.

Stack полностью разрешается перед переходом к следующему этапу.

---

# State-Based Actions

После каждого разрешения стека:

- проверяется смерть игроков;
- проверяется смерть монстров;
- проверяется уничтожение карт;
- проверяется получение душ;
- проверяются победные условия.

Если появились новые изменения —
цикл начинается заново.

---

# Завершение цикла

Когда:

- Stack пуст;
- Event Queue пуста;
- SBA отсутствуют;

движок возвращается в режим ожидания следующей команды.
